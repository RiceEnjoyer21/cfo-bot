Deadline: Week 10 Teams: Groups of maximum 5 or solo Total Points: 16																
																
Project Overview																
																
Chat Bot application																
																
For this project, your group will build and deploy a CFO Bot, a calculator agent that estimates monthly cloud costs based on usage assumptions and model selection.
However, you will not simply open an IDE and start "vibe coding." As senior Information Systems students, 
you will utilize Spec-Driven Development (SDD) via Google Antigravity to architect, verify, and deploy your application to a live Google Firebase environment.																
																
Project Phases & Deliverables																
																
Phase 1: The System Specification (The SSOT) for  Chat Bot application. Before any code is generated, your group must act as requirements engineers.
You will write a strict, highly structured specification document that serves as the Single Source of Truth (SSOT). This is a contract, not a prompt. It must clearly define:																
																
The specific cloud components your bot will support (e.g., Compute, Storage, Bandwidth, Database tiers).																
The exact mathematical cost models and pricing logic the bot must apply.																
Architectural constraints and UI/UX requirements for the final web app.																
																
Phase 2: Antigravity Orchestration Using Google Antigravity’s Agent Manager, you will feed your SSOT to the agents.
You are responsible for orchestrating these agents to produce verifiable artifacts before the actual application code is finalized.																
																
Required Deliverables: An AI-generated Implementation Plan and Test Specifications to guarantee the calculator's math is flawlessly accurate.																
																
Phase 3: Firebase Deployment The final frontend interface (and any serverless backend logic required) must be deployed, functional, and publicly accessible using Google Firebase.																
																
Phase 4: Pricing Strategy Document Using the outputs and unit economics generated by your CFO Bot, write a cohesive business strategy document justifying your selected cloud architecture.																
																
Defense & Assessment Rubric (16 Points)																
																
You will defend your project in class. Every member of the group must be prepared to explain why specific architectural choices were made and how the agents were orchestrated.
If you cannot explain your codebase, you will lose points during the defense.																
																
																
Criteria	Description	Points														
1. Specification Quality	The System Spec acts as a true Single Source of Truth, clearly defining scope, mathematical models, and system constraints.	/4														
2. SDD Workflow & Artifacts	The group successfully managed Antigravity agents to generate tangible artifacts (Implementation Plans, Test Specs) rather than relying on trial-and-error coding.	/4														
3. Application & Firebase Hosting	The CFO Bot accurately calculates cloud costs, handles edge cases gracefully, and is successfully deployed to a live Firebase URL.	/4														
4. Defense & Pricing Strategy	The group confidently defends the code structure and agent orchestration. The AI-generated pricing document is logical, accurate, and business-ready.	/4
